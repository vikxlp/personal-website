# Postcards — Feature Documentation

## What this is

A personal, low-pressure section of the site for casual updates — what I'm doing, eating, watching, reading, listening to, thinking about. Written for friends and acquaintances, not the public internet.

---

## Intent, purpose, audience, tone

**Intent:** Stay in touch with people who care, without the pressure of a product or schedule.

**Purpose:** A personal log of life — text, images with captions, links — published on the site and delivered optionally via email or RSS.

**Audience:** Friends, acquaintances, people who explicitly opted in. Not strangers, not SEO.

**Tone:** Casual, human, first-person. Like a group email to close friends — no growth-optimization, no open-rate anxiety. Publish when there's something to say.

---

## Technical structure

```
URL:      /postcards/
Feed:     /postcards/index.xml   (RSS 2.0, full content)

Content:  content/postcards/*.md
Layout:   layouts/postcards/list.html     (listing with subscribe section)
          layouts/postcards/single.html   (cover image, hidden title)
Feed:     layouts/postcards/rss.xml
Archetype: archetypes/postcards.md
Script:   new-postcard.sh               (auto-generates filename)

Images:   static/images/postcards/      (cover images)
          static/images/profile.png     (144×144 — RSS channel icon)
```

**Home page:** Postcards section appears automatically when at least one is published.

**Navigation:** Not in the main nav. Discoverable via home page or direct link.

**Feed discovery:** `<link rel="alternate">` and `<link rel="apple-touch-icon">` injected in `<head>` on postcards pages for feed reader auto-detection.

---

## RSS feed

Feed URL: `/postcards/index.xml` — RSS 2.0 with Media RSS namespace.

**Channel-level:** Title "Postcards from Vikalp", channel image (`profile.png` at 144×144).

**Per item:**
- Title: date formatted as `D Mon YYYY` (e.g. `25 Feb 2026`)
- Description: cover image (if set) as `<figure><img>` + full postcard HTML
- `<media:thumbnail>` + `<media:content>`: cover image URL for feed reader thumbnails

## Display behaviour

| Surface | What shows |
|---|---|
| `/postcards/` listing | Date only — no title, no summary |
| Home page postcard cards | Date only |
| Single postcard page | Cover image (optional) + date + body. Title hidden. |
| RSS item title | Date in `D Mon YYYY` format |
| RSS item description | Cover image (if set) + full HTML content |
| Footer | Hidden on single postcards; visible on listing |

## Content model

Filename format: `N-YYYY-MM-DD.md` — auto-generated by `new-postcard.sh`.

**Front matter:**
```yaml
---
title: "1"           # Chronological number, starting from 1
date: YYYY-MM-DD
draft: false
cover: "/images/postcards/01-descriptor.jpg"   # optional
coverAlt: "Alt text"                           # optional, used in RSS
coverCaption: "Caption below image"            # optional, site only
---
```

**Cover image naming:** `NN-descriptor.jpg` (e.g. `01-jawai-sunrise.jpg`) in `static/images/postcards/`.

**Inline image with caption:**
```html
<figure>
  <img src="/images/postcards/photo.jpg" alt="description" loading="lazy">
  <figcaption>A short caption here.</figcaption>
</figure>
```

**Supported content:** Markdown paragraphs, `### headings` (h3), bold/italic, links, blockquotes, lists, raw HTML figures.

## Workflow

```bash
bash new-postcard.sh          # creates content/postcards/N-YYYY-MM-DD.md
# edit: set draft: false, add cover if needed, write content
hugo server -D                # preview at /postcards/ and /postcards/N-YYYY-MM-DD/
git add content/postcards/ static/images/postcards/
git commit -m "Postcard N" && git push
```

**RSS local testing note:** Drafts are excluded from RSS even with `-D`. Images in RSS appear blank in readers when served from localhost — readers sandbox local requests. Use `ngrok http 1313` for a public URL to test images in a reader.

## Newsletter (Buttondown)

The `/postcards/` page has an email subscribe form. Email delivery is manual via [Buttondown](https://buttondown.email) (free, up to 100 subscribers).

**Setup:** Replace `BUTTONDOWN_USERNAME` in `layouts/postcards/list.html` with your username.

**Per postcard:** Publish to site → log into Buttondown → send email linking to the postcard URL. Manual, ~2 minutes.

**RSS:** `/postcards/index.xml` works independently. The subscribe form is the only Buttondown dependency.
