# Postcards — Feature Documentation

## What this is

A personal, low-pressure section of the site for casual updates — what I'm doing, eating, watching, reading, listening to, thinking about. Written for friends and acquaintances, not the public internet.

Not a newsletter in the marketing sense. Just postcards.

---

## Intent, purpose, audience, tone

**Intent:** Stay in touch with people who care, without the pressure of a product or schedule.

**Purpose:** A personal log of life — text, images with captions, links — published on the site and delivered optionally via email or RSS.

**Audience:** Friends, acquaintances, people who explicitly opted in. Not strangers, not SEO.

**Tone:** Casual, human, first-person. Like a group email to close friends — not a broadcast. No growth-optimization. No open-rate anxiety. Publish when there's something to say.

## Technical structure

```
URL:      /postcards/
Feed:     /postcards/index.xml   (RSS 2.0, full content)

Content:  content/postcards/*.md
Layout:   layouts/postcards/list.html     (listing with subscribe section)
          layouts/postcards/single.html   (postcard-specific: cover image, hidden title)
Feed:     layouts/postcards/rss.xml
Archetype: archetypes/postcards.md
Script:   new-postcard.sh               (auto-generates filename, runs hugo new)

Images:   static/images/postcards/      (cover images)
          static/images/profile.png     (144×144 — RSS channel icon / apple-touch-icon)
```

**Home page:** A "Postcards" section appears automatically when at least one postcard is published. Hidden when empty.

**Navigation:** Not in the main nav. Discoverable via the home page or a direct link.

**Feed discovery:** `<link rel="alternate">` tags are injected in `<head>` for any postcards page, so feed readers auto-detect the feed. A `<link rel="apple-touch-icon">` pointing to `profile.png` is also injected on postcards pages — Reeder and NetNewsWire use this to set the feed's icon.

## RSS feed

Feed URL: `/postcards/index.xml` — RSS 2.0 with Media RSS namespace.

**Channel-level:**
- Title: "Postcards from Vikalp"
- Channel image (`<image>`): `profile.png` at 144×144. Used by spec-compliant readers as the feed icon.
- Apple touch icon: `profile.png` served via `<link rel="apple-touch-icon">` on the postcards pages — Reeder/NetNewsWire prefer this over the RSS `<image>` element for the feed icon.

**Per item:**
- Title: date formatted as `D Mon YYYY` (e.g. `25 Feb 2026`)
- Description: cover image (if set) prepended as `<figure><img>` + full postcard HTML content
- `<media:thumbnail>`: cover image URL — used by Reeder for the article list thumbnail
- `<media:content>`: cover image URL — full media attachment

The cover image in the description is populated from the `cover` and `coverAlt` front matter fields. If no cover is set, only the article body appears in the description.

**Footer:** Hidden on single postcard pages (`/postcards/N-YYYY-MM-DD/`). Visible on the listing page.

## Content model

Each postcard is a Markdown file in `content/postcards/`.

**Filename format:** `N-YYYY-MM-DD.md` — auto-generated by `new-postcard.sh` (see below).

**Front matter:**
```yaml
---
title: "1"           # Chronological number, starting from 1
date: YYYY-MM-DD
draft: false
cover: "/images/postcards/01-descriptor.jpg"       # optional cover image
coverAlt: "Alt text for the image"                 # optional, used in RSS too
coverCaption: "Caption shown below the image"      # optional, shown on site only
---
```

**Cover image naming:** `NN-descriptor.jpg` prefixed with the zero-padded postcard number (e.g. `01-jawai-sunrise.jpg`). Images go in `static/images/postcards/`.

**Supported content:**

| Type | How to write |
|------|-------------|
| Text | Standard Markdown paragraphs |
| Heading | `### Section` (h3) |
| Bold/italic | `**bold**`, `_italic_` |
| Links | `[text](url)` |
| Image with caption | Raw HTML `<figure>` block (see below) |
| Blockquote | `> text` |
| List | `- item` |

**Inline image with caption (HTML, since Markdown doesn't support `figcaption`):**
```html
<figure>
  <img src="/images/postcards/photo.jpg" alt="description" loading="lazy">
  <figcaption>A short caption here.</figcaption>
</figure>
```
Images go in `static/images/postcards/`. Use descriptive `alt` text. Always include `loading="lazy"`.

## How to write a postcard

### 1. Create the file

Run the script from the project root — it auto-names the file `N-YYYY-MM-DD.md`:

```bash
bash new-postcard.sh
```

**What the script does:**
- Counts existing postcards in `content/postcards/` to determine the next number
- Gets today's date
- Runs `hugo new postcards/N-YYYY-MM-DD.md` for you

Example output: `→ content/postcards/1-2026-02-25.md`

### 2. Edit front matter

Open the created file and set:
```yaml
title: "N"           # the number
date: YYYY-MM-DD     # already set to today
draft: false
```

Add `cover`, `coverAlt`, `coverCaption` if this postcard has a cover image.

### 3. Write in Markdown

Write freely. No word count, no structure required. If adding images, put them in `static/images/postcards/` and use the `<figure>` block above.

### 4. Preview locally
```bash
hugo server -D
# Visit http://localhost:1313/postcards/           → listing
# Visit http://localhost:1313/postcards/1-YYYY-MM-DD/  → single postcard
# Visit http://localhost:1313/postcards/index.xml  → RSS feed
```

**Note:** Drafts are excluded from RSS even with `-D`. Set `draft: false` to test the RSS feed locally. Image URLs in the RSS will use `http://localhost:1313/` when served locally — they switch to the production domain on deploy. RSS readers (including Reeder) will load the feed from localhost but **images won't render** — profile picture and cover image appear blank because readers sandbox localhost image requests. To test images in a reader, either push to production or use `ngrok http 1313` to get a public HTTPS URL.

### 5. Publish
```bash
git add content/postcards/N-YYYY-MM-DD.md static/images/postcards/
git commit -m "Postcard N"
git push
```
Netlify auto-deploys. The postcard appears at `/postcards/N-YYYY-MM-DD/`, in the RSS feed, and on the home page.

## Newsletter integration (Buttondown — free, manual sends)

The `/postcards/` page has an email subscribe form and RSS link. Email delivery is handled by [Buttondown](https://buttondown.email) on the free tier.

**On RSS-to-email automation:** It's a paid feature on every major platform — Buttondown ($9/mo), Beehiiv (Max plan), and Kit ($39/mo). Not worth paying for a casual personal list. Manual sends take 2 minutes.

### First-time setup

1. Create a Buttondown account at [buttondown.email](https://buttondown.email) (free, up to 100 subscribers)
2. Choose a username (e.g. `vikalpgupta`)
3. In [layouts/postcards/list.html](layouts/postcards/list.html), replace `BUTTONDOWN_USERNAME` with your actual username
4. Push and deploy — the subscribe form at `/postcards/` is now live

### Workflow per postcard

1. Publish the postcard (push to main → Netlify deploys)
2. Log into Buttondown → New Email
3. Write the email — paste the postcard content, or simply link to `vikalpgupta.com/postcards/N-YYYY-MM-DD/`
4. Send to subscribers

### RSS (independent of Buttondown)

The feed at `/postcards/index.xml` works without Buttondown. Readers can subscribe in any feed reader (Feedly, NetNewsWire, Reeder, Miniflux). The subscribe form is the only Buttondown dependency.
